Open ALMA

Open ALMA is a web based enterprise letters system particularly suited for large hospitals. Currently under development, the software will be released under a BSD open source license when completed.

The system will interface with any hospital PAS using standard API calls and will drastically reduce the time it takes to produce letters by pulling patient demographics and registered GP information directly from the main hospital system.

Open ALMA has a sophisticated work-flow system which directs letters quickly to internal departments such as waiting list and clinical coding to reduce the movement of paper. Consultants and health professionals will be able to check and digitally sign letters on-line reducing the turn around time for issuing letters electronically to GP’s.
— Paul Bradley (lead developer)

System Requirements

The system has been designed with speed & stability as two key features. As a result Open ALMA runs on very modest hardware requirements thanks to the technology stack chosen.

Server Technology

  1. PostgreSQL — highly optimised enterprise level database
  2. Apache — webserver
  3. PHP — scripting language
  4. RabbitMQ - highly reliable enterprise messaging system
  5. Memcached
  6. LaTeX — typesetting system to produce high quality letters

Client Technology

  1. A decent browser, Google Chrome, Firefox or IE10+
  2. autohotkey — used for autotext / auto completion.

Open ALMA Consortium

The Open ALMA Consortium is a membership association dedicated to insuring the continuing vitality and independence of the software. Consortium members contribute funding to support the ongoing development and maintenance of the code base and in return receive enterprise-level technical support and legal assurances that Open ALMA will remain independent and open-source.

Although not required, consortium membership entitles you to:

  • Technical support & implementation consultation
  • Custom changes as required
  • License to use the clinical spell checker
  • Nightly encrypted off site backups
  • Printed manuals for your secretaries
  • Performance management and software analytics via New Relic
  • Notification of service issues before your users are affected by integrating into Raygun
  • Software as a service. Don’t have the necessary Linux skills to set-up & maintain the server then we can run the system as a complete service over a secure connection.
